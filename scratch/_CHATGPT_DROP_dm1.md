# Q232-dm1: `normEDS` as an elliptic sequence and the minimal Somos instances

## Executive answer

The full theorem

```lean
theorem isEllSequence_normEDS (b c d : R) : IsEllSequence (normEDS b c d)
```

is Ward's theorem for the normalised EDS generated by `(b,c,d)`.  It is concrete, but it is **not** a one-screen consequence of `normEDS_even`/`normEDS_odd`.  A direct proof needs a simultaneous induction over a family of addition laws, or Ward's EDS-net argument.  The current Mathlib file explicitly defines `IsEllSequence`, `normEDS`, `complEDS`, and the even/odd recurrences, but its own TODO says to prove that `normEDS` satisfies `IsEllDivSequence`.

For the division-polynomial doubling identity, the full theorem is overkill.  The minimal lemma you actually need is the adjacent Somos relation, the `(m,2,1)` instance of `IsEllSequence`:

```lean
W_{m+2} W_{m-2} = W_{m+1} W_{m-1} W_2^2 - W_3 W_m^2.
```

For `normEDS b c d`, since `W_2 = b` and `W_3 = c`, this is:

```lean
normEDS b c d (m + 2) * normEDS b c d (m - 2)
  = normEDS b c d (m + 1) * normEDS b c d (m - 1) * b^2
      - c * normEDS b c d m ^ 2
```

For Mathlib's **pre**-normalised division polynomials, where even terms omit one factor of `ψ₂`, this becomes the parity-normalised relation:

```lean
preΨ(m+2) * preΨ(m-2)
  = (if Even m then 1 else Ψ₂Sq^2) * preΨ(m+1) * preΨ(m-1)
      - preΨ(3) * preΨ(m)^2
```

The sign is **minus**, not plus.  This is important.  It is the specialization

```text
ψ_{m+2} ψ_{m-2} ψ_1²
  = ψ_{m+1} ψ_{m-1} ψ_2² - ψ_3 ψ_1 ψ_m².
```

## 1. Which proof strategy is Lean-feasible?

### Full Ward theorem

The full theorem

```lean
IsEllSequence (normEDS b c d)
```

should be proved by a **simultaneous induction on a rectangular family of addition formulas**, not by a naive induction on `(m,n,r)`.  The recurrence only gives formulas for doubling and adjacent odd indices:

```lean
normEDS_even
normEDS_odd
```

Those determine the sequence, but the full three-variable relation is not structurally aligned with a single recursion variable.

A feasible full proof architecture is:

```lean
/-- Addition law with fixed gap. -/
def AddRel (W : ℤ → R) (a b : ℤ) : Prop :=
  ∀ m : ℤ,
    W (m + a) * W (m - a) * W b ^ 2
      = W (m + b) * W (m - b) * W a ^ 2
        - W (a + b) * W (a - b) * W m ^ 2
```

Then prove a closure theorem saying that if `AddRel` holds for enough smaller gaps, it holds after doubling and after the odd recurrence.  This is essentially Ward's EDS-net proof in one dimension.

The simpler idea “prove `r=1` and `r=2`, then derive all `r` by induction using the relation itself” is mathematically fine on paper, but in Lean it tends to become circular unless the induction invariant already contains enough addition laws to justify the recurrence step.  The relation itself is what one is trying to prove.

### Minimal route for the current downstream proof

For the doubling identity, prove only:

```lean
normEDS_adjacent_somos
```

or, for division polynomials,

```lean
preΨ_adjacent_somos
```

This is the smallest practical theorem.  It can be proved either:

1. from the future full theorem `isEllSequence_normEDS`; or
2. by a dedicated simultaneous induction on `m` using the even/odd recurrences, with generated `ring` certificates for the parity steps.

I recommend option 2 only if you want the doubling identity closed before formalising the full Ward theorem.

## 3. Direct proof skeleton for the minimal adjacent relation

The dedicated proof should not try to prove the full `IsEllSequence`.  Prove only the invariant

```lean
def AdjRel (b c d : R) (m : ℤ) : Prop :=
  normEDS b c d (m + 2) * normEDS b c d (m - 2)
    = normEDS b c d (m + 1) * normEDS b c d (m - 1) * b ^ 2
        - c * normEDS b c d m ^ 2
```

Because `normEDS` is odd, `AdjRel` is invariant under `m ↦ -m`.  So reduce to nonnegative `m`, then do strong induction on `m : ℕ`.

```lean
namespace FLT.EDS

variable {R : Type*} [CommRing R]

def AdjRel (b c d : R) (m : ℤ) : Prop :=
  normEDS b c d (m + 2) * normEDS b c d (m - 2)
    = normEDS b c d (m + 1) * normEDS b c d (m - 1) * b ^ 2
        - c * normEDS b c d m ^ 2

lemma AdjRel_neg (b c d : R) (m : ℤ) :
    AdjRel b c d (-m) ↔ AdjRel b c d m := by
  unfold AdjRel
  simp [normEDS_neg]
  ring_nf

lemma AdjRel_zero (b c d : R) : AdjRel b c d 0 := by
  unfold AdjRel
  simp [normEDS_zero, normEDS_one, normEDS_two, normEDS_neg]
  ring

lemma AdjRel_one (b c d : R) : AdjRel b c d 1 := by
  unfold AdjRel
  simp [normEDS_zero, normEDS_one, normEDS_two, normEDS_three, normEDS_neg]
  ring

lemma AdjRel_two (b c d : R) : AdjRel b c d 2 := by
  unfold AdjRel
  simp [normEDS_zero, normEDS_one, normEDS_two, normEDS_three, normEDS_four]
  ring

lemma AdjRel_three (b c d : R) : AdjRel b c d 3 := by
  unfold AdjRel
  -- This uses `normEDS_five`, which unfolds via `normEDS_odd 2`.
  rw [show (3 + 2 : ℤ) = 2 * 2 + 1 by ring]
  rw [normEDS_odd b c d 2]
  simp [normEDS_one, normEDS_two, normEDS_three, normEDS_four]
  ring

/-- Even induction step for the adjacent relation. -/
lemma AdjRel_even_step
    (b c d : R) (k : ℤ)
    (hk_m2 : AdjRel b c d (k - 2))
    (hk_m1 : AdjRel b c d (k - 1))
    (hk    : AdjRel b c d k)
    (hk_p1 : AdjRel b c d (k + 1))
    (hk_p2 : AdjRel b c d (k + 2)) :
    AdjRel b c d (2 * k) := by
  unfold AdjRel at *
  -- Expand `normEDS (2*k + j)` for j=-2,-1,0,1,2 using `normEDS_even`/`normEDS_odd`.
  -- Then reduce using the five local adjacent relations around k.
  -- This is a large but finite `ring` certificate.
  -- The reliable implementation is generated by `linear_combination` over hk_m2..hk_p2.
  sorry

/-- Odd induction step for the adjacent relation. -/

    (b c d : R) (k : ℤ)
    (hk_m2 : AdjRel b c d (k - 2))
    (hk_m1 : AdjRel b c d (k - 1))
    (hk    : AdjRel b c d k)
    (hk_p1 : AdjRel b c d (k + 1))
    (hk_p2 : AdjRel b c d (k + 2)) :
    AdjRel b c d (2 * k + 1) := by
  unfold AdjRel at *
  -- Same as the even step, using `normEDS_odd` for the central term.
  sorry

/-- Dedicated proof of the adjacent Somos relation. -/
theorem normEDS_adjacent_somos_by_induction
    (b c d : R) (m : ℤ) : AdjRel b c d m := by

end FLT.EDS
```

The only nontrivial sublemmas are `AdjRel_even_step` and `AdjRel_odd_step`.  They are not conceptually hard, but the expanded terms are large.  I recommend generating their ideal certificates mechanically, exactly like for the division-polynomial doubling residual.

## 4. How to generate the even/odd step certificates

For the even step, introduce symbolic variables

```text
A_i = W(k+i),   i = -4,-3,-2,-1,0,1,2,3,4
b,c,d as coefficient parameters
```

Relations are the local adjacent laws for `k-2`, `k-1`, `k`, `k+1`, `k+2`:

```text
A_{i+2} A_{i-2} - A_{i+1} A_{i-1} b^2 + c A_i^2 = 0
```

for `i = -2,-1,0,1,2`.

Then encode `W(2k+j)` using `normEDS_even`/`normEDS_odd` in terms of the `A_i`.  Form the residual for `AdjRel(2k)` or `AdjRel(2k+1)`, reduce it modulo the five relations, and extract cofactors.

Sage sketch:

```python
# sage
R = PolynomialRing(QQ, names=['b','c'] + [f'A{i}' for i in range(-5,6)], order='degrevlex')
b, c, *As = R.gens()
A = {i: As[i+5] for i in range(-5,6)}

def Adj(i):
    return A[i+2]*A[i-2] - A[i+1]*A[i-1]*b^2 + c*A[i]^2

rels = [Adj(i) for i in [-2,-1,0,1,2]]
I = ideal(rels)

# Define E(j) = W(2k+j) by the normEDS recurrences.
# For example:
# W(2u) = W(u-1)^2 W(u) W(u+2) - W(u-2) W(u) W(u+1)^2
# W(2u+1) = W(u+2) W(u)^3 - W(u-1) W(u+1)^3
# with the normalised b-factors already present in normEDS_even/odd.

def W_even(u):
    return A[u-1]^2*A[u]*A[u+2] - A[u-2]*A[u]*A[u+1]^2

def W_odd(u):
    return A[u+2]*A[u]^3 - A[u-1]*A[u+1]^3

# Fill in exact normalised `normEDS_odd` b-factors if the Mathlib recurrence has them.
# Then residual for AdjRel(2k):
res_even = W_even(1)*W_even(-1) - W_odd(0)*W_odd(-1)*b^2 + c*W_even(0)^2
assert I.reduce(res_even) == 0
cofs_even = I.lift(res_even)
print([f.factor() for f in cofs_even])
```

Important: the exact normalised `normEDS_odd` recurrence in Mathlib has no parity branch after normalisation; use the statement of `normEDS_odd` from the file rather than the pre-normalised recurrence.

## 6. Practical build plan

1. Add `normEDS_adjacent_somos` as a local theorem, not the full `IsEllSequence_normEDS`.
2. Prove it by the induction skeleton above, using generated `linear_combination` certificates for `AdjRel_even_step` and `AdjRel_odd_step`.
3. Derive `preΨ_adjacent_somos` from `normEDS_adjacent_somos` by unfolding `WeierstrassCurve.preΨ` to the corresponding `preNormEDS` parameters.
4. Use `preΨ_adjacent_somos_even` / `preΨ_adjacent_somos_odd` to close the division-polynomial doubling residual.

This is the shortest path to closing the division-polynomial doubling/addition identities.  The full theorem

```lean
IsEllSequence (normEDS b c d)
```

should be left for a later Mathlib contribution once the adjacent instance is working.
