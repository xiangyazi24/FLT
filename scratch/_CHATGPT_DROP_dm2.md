# Q261 (dm2): General Weierstrass `addX` cofactor `Q₃` for `m = 3`

Target identity:

```text
addX([X,Y,1], [φ₃,ω₃,ψ₃]) - ψ₂²·φ₄ = Q₃ · F_W
```

with

```text
F_W = Y² + a₁XY + a₃Y - X³ - a₂X² - a₄X - a₆.
```

## Result

The division remainder is zero:

```text
remainder_zero = True
```

The cofactor has

```text
terms(Q₃) = 2521
Y-degree(Q₃) = 4
```

Since `terms(Q₃) > 200`, I am not printing the full expanded polynomial.

## Factorization / compact form

SymPy factors `Q₃` over `ℚ[a₁,a₂,a₃,a₄,a₆,X,Y]` as

```text
Q₃ = 2 · ψ₃ · H₃
```

where

```text
ψ₃ = 3X⁴ + b₂X³ + 3b₄X² + 3b₆X + b₈
```

and `H₃` is a large irreducible factor with

```text
terms(H₃) = 735
Y-degree(H₃) = 4
X-degree(H₃) = 12
```

Thus there is **no compact analogue** of the `m=2` factorization

```text
Q₂ = 2 · ψ₂² · Ψ₂Sq.
```

The only clean recognizable factor for `m=3` is `ψ₃`; the remaining factor `H₃` is already too large to be useful as a compact hand-written Lean expression.

Equivalently, the best compact statement found is:

```text
Q₃ = 2 · Ψ₃ · H₃,
```

with `H₃ := Q₃ / (2 Ψ₃)` generated by CAS.

## Complete runnable SymPy script

This uses SymPy’s sparse polynomial-ring backend.  A direct fully-expanded expression computation is much slower and can time out.

```python
from sympy.polys.rings import ring
from sympy.polys.domains import QQ

# Sparse polynomial ring.  The variable order puts Y first so division by F_W
# is division by a monic quadratic in Y.
R, Y, X, a1, a2, a3, a4, a6 = ring('Y,X,a1,a2,a3,a4,a6', QQ, order='lex')

FW = Y**2 + a1*X*Y + a3*Y - X**3 - a2*X**2 - a4*X - a6

b2 = a1**2 + 4*a2
b4 = 2*a4 + a1*a3
b6 = a3**2 + 4*a6
b8 = a1**2*a6 + 4*a2*a6 - a1*a3*a4 + a2*a3**2 - a4**2

psi2 = 2*Y + a1*X + a3
psi3 = 3*X**4 + b2*X**3 + 3*b4*X**2 + 3*b6*X + b8
prepsi4 = (
    2*X**6 + b2*X**5 + 5*b4*X**4 + 10*b6*X**3 + 10*b8*X**2
    + (b2*b8 - b4*b6)*X + (b4*b8 - b6**2)
)
psi4 = psi2 * prepsi4

# Actual bivariate ψ_5 and ψ_6 from the EDS recurrence.
psi5 = psi4*psi2**3 - psi3**3
psi6 = (psi3 * (psi2**2 * psi5 - psi4**2)).exquo(psi2)

phi3 = X*psi3**2 - psi4*psi2
phi4 = X*psi4**2 - psi5*psi3

# 2*ψ₃*ω₃ = ψ₆ - ψ₃²*(a₁*φ₃ + a₃*ψ₃²)
omega3 = (psi6 - psi3**2 * (a1*phi3 + a3*psi3**2)).exquo(2*psi3)


def jacobian_addX(P, Q):
    """Mathlib WeierstrassCurve.Jacobian.addX from Jacobian/Formula.lean."""
    P0, P1, P2 = P
    Q0, Q1, Q2 = Q
    return (
        P0*Q0**2*P2**2
        - 2*P1*Q1*P2*Q2
        + P0**2*Q0*Q2**2
        - a1*P0*Q1*P2**2*Q2
        - a1*P1*Q0*P2*Q2**2
        + 2*a2*P0*Q0*P2**2*Q2**2
        - a3*Q1*P2**4*Q2
        - a3*P1*P2*Q2**4
        + a4*Q0*P2**4*Q2**2
        + a4*P0*P2**2*Q2**4
        + 2*a6*P2**4*Q2**4
    )


P = [X, Y, R(1)]
R3 = [phi3, omega3, psi3]

addX3 = jacobian_addX(P, R3)
bigPoly = addX3 - psi2**2 * phi4

# Divide by F_W.  Since F_W is monic in Y, the quotient is canonical.
qs, rem = bigPoly.div([FW])
Q3 = qs[0]

print('remainder_zero =', rem == 0)
print('terms(Q3) =', len(Q3.terms()))
print('degree_Y(Q3) =', Q3.degree(Y))

# Factor summary.  Full factor printing is intentionally suppressed because H3 is large.
fac = Q3.factor_list()
print('factor_constant =', fac[0])
print('number_of_nonconstant_factors =', len(fac[1]))
for i, (factor, exp) in enumerate(fac[1], start=1):
    print(f'factor_{i}_exp =', exp)
    print(f'factor_{i}_terms =', len(factor.terms()))
    print(f'factor_{i}_degree_Y =', factor.degree(Y))
    print(f'factor_{i}_degree_X =', factor.degree(X))
    print(f'factor_{i}_is_psi3 =', factor == psi3)

H3 = Q3.exquo(2*psi3)
print('H3_terms =', len(H3.terms()))
print('H3_degree_Y =', H3.degree(Y))
print('H3_degree_X =', H3.degree(X))
print('Q3_equals_2_psi3_H3 =', Q3 == 2*psi3*H3)

# Explicit output only if small.
if len(Q3.terms()) < 200:
    print('Q3_expanded =')
    print(Q3)
else:
    print('Q3_expanded omitted: term count >= 200')
```

## Lean-facing recommendation

For Lean, I would not hand-write `Q₃` expanded.  Use a generated definition for `H₃` or `Q₃`, or avoid exposing `Q₃` directly by proving the coordinate-ring identity via a generated cofactor lemma.

The compact theorem statement can use:

```lean
def addX_m3_Q₃ (W : WeierstrassCurve K) : K[X][Y] :=
  (2 : K[X][Y]) * C W.Ψ₃ * addX_m3_H₃ W
```

where `addX_m3_H₃` is the CAS-generated 735-term polynomial.  This is much better than a 2521-term flat `Q₃`, but it is still not a small mathematical factorization like the `m=2` case.
