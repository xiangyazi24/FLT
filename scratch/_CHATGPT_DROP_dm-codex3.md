# Q2804: route audit for E1 affine rational points

Curve:

```text
E1 : y^2 = x (x - 1) (x + 3) = x^3 + 2*x^2 - 3*x
```

Desired affine classification:

```text
x ∈ {-3, 0, 1, -1, 3}
points: (-3,0), (0,0), (1,0), (-1,±2), (3,±6)
```

## Route selection verdict

Choose **A: full-cover extraction**.

The 2-isogeny descent certificate is mathematically clean, but in Lean/mathlib today it has much higher infrastructure risk: it needs elliptic-curve group laws, isogenies, dual-isogeny exactness, Selmer-to-rank formula, and torsion enumeration. Your full-cover route has already checked the hard finite-cover enumeration and the downstream eliminations. The remaining full-cover extraction is arithmetic plumbing over `ℚ` and `ℤ`; hard but local, with no elliptic-curve group infrastructure.

The clean DAG is:

```text
rational affine point on E1, Y≠0
  -> squareclasses of x, x-1, x+3 supported only at {2,3}
  -> choose d0,d1,d3 ∈ {±1,±2,±3,±6}, product squareclass
  -> clear denominators to primitive CoverInt tuple A,B,C,T
  -> checked cover enumeration/elimination
  -> contradiction for Y≠0 except x=-1,3
  -> add Y=0 roots x=-3,0,1
  -> affine classification
```

## 1. Explicit 2-isogeny certificate, for reference

Let

```text
E  : y^2 = x^3 + 2*x^2 - 3*x
E' : y^2 = x^3 - 4*x^2 + 16*x
```

This is the standard 2-isogeny with kernel generated by `(0,0)` on
`E : y^2 = x^3 + a*x^2 + b*x`, where `a=2`, `b=-3`.

Forward isogeny:

```text
φ : E -> E'
φ(x,y) = (y^2/x^2, y*(x^2 + 3)/x^2)        for x≠0
φ(0,0) = O, φ(O)=O
```

because `x^2-b = x^2+3`.

Dual before scaling: for `E'` with `a'=-4`, `b'=16`, the standard 2-isogeny lands on

```text
E'' : y^2 = x^3 + 8*x^2 - 48*x.
```

`E''` is isomorphic to `E` by `(x'',y'') = (4*x, 8*y)`. Thus the dual map to `E` is

```text
φhat(X,Y) = (Y^2/(4*X^2), Y*(X^2-16)/(8*X^2))      for X≠0.
```

### Isogeny descent covers

For `E : y^2 = x^3 + a*x^2 + b*x`, the homogeneous spaces for the isogeny with kernel `(0,0)` are

```text
C_d : N^2 = d*M^4 + a*M^2*R^2 + (b/d)*R^4
```

with `d` representing a squareclass divisor of `b`.

For `E`, `a=2`, `b=-3`:

```text
C_d : N^2 = d*M^4 + 2*M^2*R^2 - (3/d)*R^4,
       d ∈ {1,-1,3,-3}.
```

All four classes are soluble and correspond to the observed image of the connecting map
`α(P)=x(P) mod squares`, with the exceptional rules `α(O)=1`, `α((0,0))=b=-3`:

```text
x=1   -> d=1
x=-1  -> d=-1
x=3   -> d=3
x=0   -> d=-3
```

For `E'`, `a'=-4`, `b'=16`:

```text
C'_d : N^2 = d*M^4 - 4*M^2*R^2 + (16/d)*R^4,
       d ∈ {1,-1,2,-2}.
```

Local certificates:

```text
d=-1, -2 : real obstruction, RHS < 0 for nonzero primitive data.
d=2      : mod 16 obstruction.
           If M odd, RHS ≡ 2 or 6 mod 8, impossible for a square.
           If M even, primitive forces R odd; RHS ≡ 8 mod 16, impossible.
d=1      : soluble, e.g. X=4,Y=±8 on E'.
```

Thus the isogeny descent certificate is:

```text
#im α_E  = 4  with classes {1,-1,3,-3}
#im α_E' = 1  with class {1}
2^rank = (#im α_E * #im α_E') / 4 = 1
rank(E(Q)) = 0
```

Torsion is exactly

```text
E(Q)_tors = {O, (0,0), (1,0), (-3,0), (-1,±2), (3,±6)}
```

with `(-1,2)` and `(3,6)` both doubling to `(1,0)`, so the group is `ℤ/2 × ℤ/4`.

### Why this is not the recommended Lean route

To make the 2-isogeny route non-circular in Lean, you need all of the following:

```text
1. definitions of E(Q), E'(Q) as groups;
2. rational maps φ, φhat and proof they are dual 2-isogenies;
3. connecting maps α, α' and exactness with the homogeneous spaces C_d, C'_d;
4. finite local solubility/non-solubility certificates for C_d, C'_d;
5. rank formula from the two isogeny descent images;
6. rank zero -> every rational point is torsion;
7. torsion enumeration, e.g. Nagell-Lutz or an equivalent finite torsion proof.
```

Mathlib’s elliptic-curve group/isogeny infrastructure is not currently a low-risk dependency for this proof. This route is mathematically elegant but formalization-heavy.

## 2. Full-cover route: exact remaining arithmetic lemmas

The full-cover route starts from a rational point `(x,y)` with `y≠0` on

```text
y^2 = x*(x-1)*(x+3).
```

Because `y≠0`, the three factors are nonzero:

```text
x ≠ 0, x-1 ≠ 0, x+3 ≠ 0.
```

### 2.1 Valuation parity away from `{2,3}`

Math statement:

For any rational prime `p` with `p ≠ 2` and `p ≠ 3`, the `p`-adic valuations of each of

```text
x, x-1, x+3
```

are even.

Reason: the pairwise differences are

```text
x - (x-1) = 1
(x+3) - x = 3
(x+3) - (x-1) = 4.
```

If `p ∤ 6`, then no two of the three factors can have positive/negative `p`-adic valuation simultaneously. Since the sum of the three valuations is `2*v_p(y)`, each individual valuation is even.

Lean classification of difficulty:

```text
Genuinely hard plumbing, but local. Needs a rational valuation or normalized numerator/denominator substitute.
No elliptic-curve theory required.
```

Suggested interface:

```lean
def RatSquareclassSupportedAt23 (q : ℚ) : Prop :=
  q ≠ 0 ∧ ∃ d : ℤ, d ∈ S23 ∧ SquareclassBy q d

theorem e1_factor_squareclasses_supported_at_23
    {x y : ℚ}
    (hy : y ≠ 0)
    (hE : y ^ 2 = x * (x - 1) * (x + 3)) :
    RatSquareclassSupportedAt23 x ∧
    RatSquareclassSupportedAt23 (x - 1) ∧
    RatSquareclassSupportedAt23 (x + 3) := by
  -- hard local arithmetic lemma: valuation parity away from 2,3
  sorry
```

If avoiding a valuation API, prove it with numerator/denominator normal forms:

```text
x     = a/t
x-1   = b/t
x+3   = c/t
with gcd conditions and c-b=4t, a-b=t, c-a=3t.
```

Then for each prime `p∤6`, use Euclid-style prime divisibility to show exponents in `a,b,c,t` have the required parity. This is more verbose than valuations but implementable with existing `Nat.Prime`/`Int.Prime` lemmas.

### 2.4 Denominator clearing to primitive CoverInt

From

```text
x     = d0*r0^2
x - 1 = d1*r1^2
x + 3 = d3*r3^2
```

choose a common positive integer denominator `T` so that

```text
r0 = A/T, r1 = B/T, r3 = C/T
```

and get

```text
d0*A^2 - d1*B^2 = T^2
d3*C^2 - d0*A^2 = 3*T^2.
```

Then divide by the common gcd of `A,B,C,T` to make the tuple primitive; equations are homogeneous of degree 2, so they survive division.

Interface:

```lean
def CoverInt (d0 d1 d3 A B C T : ℤ) : Prop :=
  d0 * A ^ 2 - d1 * B ^ 2 = T ^ 2 ∧
  d3 * C ^ 2 - d0 * A ^ 2 = 3 * T ^ 2

def PrimitiveTuple4 (A B C T : ℤ) : Prop :=
  Int.gcd (Int.gcd (Int.gcd A B) C) T = 1

theorem e1_coverInt_of_squareclasses
    {x : ℚ} {d0 d1 d3 : ℤ}
    (hx0 : SquareclassBy x d0)
    (hx1 : SquareclassBy (x - 1) d1)
    (hx3 : SquareclassBy (x + 3) d3)
    (hx_ne : x ≠ 0) (hx1_ne : x - 1 ≠ 0) (hx3_ne : x + 3 ≠ 0) :
    ∃ A B C T : ℤ,
      A ≠ 0 ∧ B ≠ 0 ∧ C ≠ 0 ∧ T ≠ 0 ∧
      PrimitiveTuple4 A B C T ∧
      CoverInt d0 d1 d3 A B C T := by
  -- common denominator + primitive division
  sorry
```

Difficulty:

```text
Routine but lengthy denominator/gcd plumbing.
No hard number theory.
```

## 3. Full-cover assembly interface

This is the residual bridge you want, because your downstream finite cover and obstruction work is already checked.

```lean
def E1Curve (x y : ℚ) : Prop :=
  y ^ 2 = x * (x - 1) * (x + 3)

def E1AffineXAllowed (x : ℚ) : Prop :=
  x = -3 ∨ x = 0 ∨ x = 1 ∨ x = -1 ∨ x = 3

def E1FullCoverExtractionStatement : Prop :=
  ∀ {x y : ℚ},
    E1Curve x y → y ≠ 0 →
      ∃ d0 d1 d3 : ℤ,
        d0 ∈ S23 ∧ d1 ∈ S23 ∧ d3 ∈ S23 ∧
        ProductSquareclassCondition d0 d1 d3 ∧
        ∃ A B C T : ℤ,
          A ≠ 0 ∧ B ≠ 0 ∧ C ≠ 0 ∧ T ≠ 0 ∧
          PrimitiveTuple4 A B C T ∧
          CoverInt d0 d1 d3 A B C T
```

Then the final affine theorem follows from your checked downstream result:

```lean
def E1CoverEliminationStatement : Prop :=
  ∀ {d0 d1 d3 A B C T : ℤ},
    d0 ∈ S23 → d1 ∈ S23 → d3 ∈ S23 →
    ProductSquareclassCondition d0 d1 d3 →
    A ≠ 0 → B ≠ 0 → C ≠ 0 → T ≠ 0 →
    PrimitiveTuple4 A B C T →
    CoverInt d0 d1 d3 A B C T → False

-- plus explicit known point wrappers for x=-1 and x=3 if the cover elimination
-- only kills nonzero-Y points outside the surviving covers.

theorem E1AffineClassification_from_fullCover
    (hExtract : E1FullCoverExtractionStatement)
    (hCoverElim : E1CoverEliminationStatement)
    {x y : ℚ} (hE : E1Curve x y) :
    E1AffineXAllowed x := by
  by_cases hy : y = 0
  · -- y=0 -> x*(x-1)*(x+3)=0 -> x=-3,0,1
    unfold E1Curve at hE
    subst y
    norm_num at hE
    -- `nlinarith`/`ring_nf` plus integral-domain product zero over ℚ
    sorry
  · obtain ⟨d0,d1,d3,hd0,hd1,hd3,hprod,A,B,C,T,hA,hB,hC,hT,hprim,hcov⟩ :=
      hExtract hE hy
    exact (hCoverElim hd0 hd1 hd3 hprod hA hB hC hT hprim hcov).elim
```

In your actual project, `hCoverElim` is not pure false for all four surviving covers; it is “surviving covers imply AP/double-leg degeneracy, contradiction for nonzero `Y`.” Use the already checked `E1FullCoverIntDataEliminationStatement` or its final wrapper in place of the schematic `E1CoverEliminationStatement`.

## 4. If you still want a 2-isogeny residual interface

A non-circular but sufficiently strong interface would be:

```lean
def E1TwoIsogenySelmerCertificate : Prop :=
  -- isogeny descent data for E and E'
  (∀ d : ℤ, d ∈ [1,-1,3,-3] ↔ E_isogeny_cover_locally_soluble d) ∧
  (∀ d : ℤ, d ∈ [1] ↔ Edual_isogeny_cover_locally_soluble d) ∧
  -- exact 2-isogeny descent rank formula
  E_rank_eq_zero_from_isogeny_images ∧
  -- torsion enumeration
  (∀ P : E1RationalPoint, P ∈ explicitEightTorsionPoints)
```

But this is essentially a large elliptic-curve library. A smaller interface that directly implies the affine classification is:

```lean
def E1RankZeroTorsionEnumerationCertificate : Prop :=
  ∀ {x y : ℚ},
    y ^ 2 = x * (x - 1) * (x + 3) →
      (x = -3 ∧ y = 0) ∨ (x = 0 ∧ y = 0) ∨ (x = 1 ∧ y = 0) ∨
      (x = -1 ∧ (y = 2 ∨ y = -2)) ∨
      (x = 3 ∧ (y = 6 ∨ y = -6))
```

This is not circular mathematically if backed by an external isogeny/rank/torsion proof, but formalization-wise it is no better than assuming the target theorem. I do **not** recommend this route unless the project decides to import/develop a substantial elliptic-curve group library.

## Bottom line

The 2-isogeny route is mathematically short but Lean-large. The full-cover extraction route is mathematically more pedestrian but Lean-local, and it connects directly to the already checked finite cover enumeration and AP/double-leg eliminations. The best next move is to formalize full-cover extraction, starting with the valuation parity / squareclass support lemma for the three factors of `x(x-1)(x+3)`.
