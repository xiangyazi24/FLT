# Q: FLT N12 shifted curve rational-point theorem Lean roadmap

## Verdict

The stated affine rational point list for

```text
F : Y^2 = X^3 + 2*X^2 - 3*X = X*(X-1)*(X+3)
```

is correct:

```text
(-3,0), (0,0), (1,0), (-1,±2), (3,±6).
```

Together with the point at infinity, these eight points form

```text
F(ℚ) ≃ ℤ/2ℤ × ℤ/4ℤ.
```

A convenient choice of generators is

```text
T = (0,0),       order 2,
P = (-1,2),      order 4,
2P = (1,0),
P + T = (3,6),
2P + T = (-3,0).
```

The shortest classical proof route suitable for Lean is a **specialized 2-isogeny descent**, not a generic Mordell-Weil/rank black box. The curve has full rational 2-torsion, and the kernel generated by `(0,0)` gives the 2-isogenous curve

```text
F' : Y^2 = X^3 - 4*X^2 + 16*X.
```

The descent images are exactly

```text
δ(F(ℚ))  = {1, -1, 3, -3}       in ℚ*/ℚ*²,
δ'(F'(ℚ)) = {1}.                in ℚ*/ℚ*².
```

The nontrivial dual candidate is squareclass `2`, and it is killed by the already-identified local obstruction

```text
W^2 = 2*m^4 - 4*m^2*n^2 + 8*n^4,
gcd(m,n)=1,
```

modulo `16`.

The Lean plan should make every finite step explicit: squareclass support, the mod-16 obstruction, the isogeny exactness kernels, the resulting `F(ℚ)/2F(ℚ)` enumeration, and then a finite torsion/reduction enumeration to recover the affine list.

## 1. Sanity check of the listed points

Substitution is immediate:

```text
X=-3:  (-3)*(-4)*0 = 0,
X=0:   0,
X=1:   0,
X=-1:  (-1)*(-2)*2 = 4,
X=3:   3*2*6 = 36.
```

For the group computations on the generalized Weierstrass model

```text
y^2 = x^3 + a2*x^2 + a4*x + a6,
a2=2, a4=-3, a6=0,
```

the tangent slope at `P=(-1,2)` is

```text
λ = (3*x^2 + 2*a2*x + a4)/(2*y)
  = (3 - 4 - 3)/4
  = -1.
```

Thus

```text
x(2P) = λ^2 - a2 - 2*x = 1 - 2 + 2 = 1,
y(2P) = -y - λ*(x(2P)-x) = -2 - (-1)*2 = 0,
```

so `2P=(1,0)`. Hence `P` has order `4`. The chord through `P=(-1,2)` and `T=(0,0)` has slope `-2`, giving

```text
P+T = (3,6).
```

The negatives are obtained by changing the sign of `Y`.

A paste-safe Lean sanity check for membership is:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

private def F_on (X Y : ℚ) : Prop :=
  Y ^ 2 = X ^ 3 + 2 * X ^ 2 - 3 * X

example : F_on (-3) 0 := by norm_num [F_on]
example : F_on 0 0 := by norm_num [F_on]
example : F_on 1 0 := by norm_num [F_on]
example : F_on (-1) 2 := by norm_num [F_on]
example : F_on (-1) (-2) := by norm_num [F_on]
example : F_on 3 6 := by norm_num [F_on]
example : F_on 3 (-6) := by norm_num [F_on]

end MazurProof.RationalPointsN12
```

## 2. Exact 2-isogeny objects

Put the curve in the standard form

```text
E_ab : y^2 = x^3 + a*x^2 + b*x,
a = 2,
b = -3.
```

The 2-isogenous curve attached to the kernel generated by `(0,0)` is

```text
E'_ab : y^2 = x^3 - 2*a*x^2 + (a^2 - 4*b)*x,
```

hence here

```text
F' : Y^2 = X^3 - 4*X^2 + 16*X.
```

For affine points with nonzero `x`, the isogeny is

```text
φ : F -> F'
φ(x,y) = ( y^2/x^2,
           y*(b - x^2)/x^2 )
       = ( y^2/x^2,
           y*(-3 - x^2)/x^2 ).
```

The special values are

```text
φ(O) = O,
φ((0,0)) = O.
```

The dual isogeny is the scaled standard dual

```text
φ' : F' -> F
φ'(X,Y) = ( Y^2/(4*X^2),
            Y*(16 - X^2)/(8*X^2) )
```

for `X ≠ 0`, with

```text
φ'(O)=O,
φ'((0,0))=O.
```

The identities to prove by `field_simp` + `ring` are:

```text
φ(P) lies on F',
φ'(Q) lies on F,
φ' ∘ φ = [2] on F,
φ ∘ φ' = [2] on F',
ker φ  = {O,(0,0) on F},
ker φ' = {O,(0,0) on F'}.
```

These formulas are much shorter to formalize for this one curve than to build a general isogeny framework first.

## 3. Kummer maps and homogeneous spaces

For

```text
E : y^2 = x^3 + a*x^2 + b*x,
```

the usual 2-isogeny descent map is

```text
δ_E : E(ℚ) -> ℚ*/ℚ*²
δ_E(O)       = 1,
δ_E((0,0))   = b,
δ_E((x,y))   = x mod squares        if x ≠ 0.
```

For `F`, this is

```text
δ(O)=1,
δ((0,0))=-3,
δ((x,y))=x mod squares.
```

For `F'`, since `b'=16`,

```text
δ'(O)=1,
δ'((0,0))=16=1,
δ'((X,Y))=X mod squares.
```

The exactness statements are:

```text
ker δ  = image φ',
ker δ' = image φ.
```

These are the core algebraic descent lemmas. They can be proved by explicit coordinate algebra. For Lean, I would avoid quotient-heavy squareclass APIs initially and use squarefree integer representatives.

### Squareclass representative convention

Use a predicate like this, rather than committing early to a quotient type:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

/-- `SqRep x d m n` means `x = d*(m/n)^2` with a primitive integer fraction.
The exact squarefree predicate can be replaced by the project's preferred API. -/
def SqRep (x : ℚ) (d m n : ℤ) : Prop :=
  Squarefree d.natAbs ∧
  n ≠ 0 ∧
  Int.gcd m n = 1 ∧
  x = (d : ℚ) * (m : ℚ)^2 / (n : ℚ)^2

end MazurProof.RationalPointsN12
```

If `Squarefree d.natAbs` is awkward, replace it by a local `SquarefreeInt d` definition. The support lemmas only need the consequence that a prime cannot divide `d` twice.

## 4. Support restriction for `F`

Suppose `P=(x,y) ∈ F(ℚ)` with `x ≠ 0` and

```text
x = d*(m/n)^2,
```

where `d` is squarefree and `gcd(m,n)=1`. From

```text
y^2 = x*(x^2 + 2*x - 3)
```

one obtains

```text
d * (d^2*m^4 + 2*d*m^2*n^2 - 3*n^4) = square.
```

Since `d` is squarefree, `d` divides the second factor. Reducing that second factor modulo `d` gives

```text
-3*n^4 ≡ 0 mod d.
```

Because `gcd(d,n)=1`, this forces

```text
d ∣ 3.
```

Therefore

```text
d ∈ {1, -1, 3, -3}.
```

All four classes occur:

```text
1   : (1,0) or O,
-3  : (0,0) under the special rule, also (-3,0) as x-class,
-1  : (-1,2),
3   : (3,6).
```

So

```text
δ(F(ℚ)) = {1, -1, 3, -3}.
```

Lean interface sketch, stated as comments so this block introduces no axioms or `sorry`:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

private def F_on (X Y : ℚ) : Prop :=
  Y ^ 2 = X ^ 3 + 2 * X ^ 2 - 3 * X

/-
theorem F_sqclass_support
    {X Y : ℚ} {d m n : ℤ}
    (hF : F_on X Y)
    (hX0 : X ≠ 0)
    (hrep : SqRep X d m n) :
    d = 1 ∨ d = -1 ∨ d = 3 ∨ d = -3
-/

/-
theorem F_delta_image_exact :
    DeltaImageF = {1, -1, 3, -3}
-/

end MazurProof.RationalPointsN12
```

The proof of `F_sqclass_support` is mostly divisibility arithmetic; no elliptic-curve group law is needed.

## 5. Support restriction for `F'` and the dual local obstruction

For

```text
F' : Y^2 = X^3 - 4*X^2 + 16*X
     = X*(X^2 - 4*X + 16),
```

the quadratic factor is always positive over `ℚ`:

```text
X^2 - 4*X + 16 = (X-2)^2 + 12 > 0.
```

Thus any affine rational point with `X ≠ 0` has `X > 0`, and its squareclass representative `d` can be taken positive.

Write

```text
X = d*(m/n)^2,
```

with `d` squarefree and positive. The same support argument gives

```text
d ∣ 16,
```

so

```text
d ∈ {1,2}.
```

The `d=2` homogeneous space is exactly

```text
W^2 = 2*m^4 - 4*m^2*n^2 + 8*n^4,
gcd(m,n)=1.                                      (D2)
```

Modulo `16`, primitive parity gives:

```text
m odd,  n odd  -> RHS ≡ 2 - 4 + 8 = 6 mod 16,
m odd,  n even -> RHS ≡ 2             mod 16,
m even, n odd  -> RHS ≡ 8             mod 16,
m even, n even -> excluded by gcd(m,n)=1.
```

Squares modulo `16` are only

```text
0, 1, 4, 9.
```

So `(D2)` is impossible. Hence

```text
δ'(F'(ℚ)) = {1}.
```

Lean interface sketch:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

private def Fdual_on (X Y : ℚ) : Prop :=
  Y ^ 2 = X ^ 3 - 4 * X ^ 2 + 16 * X

/-
theorem Fdual_x_nonneg
    {X Y : ℚ} (h : Fdual_on X Y) : 0 ≤ X

theorem Fdual_sqclass_support
    {X Y : ℚ} {d m n : ℤ}
    (hF : Fdual_on X Y)
    (hX0 : X ≠ 0)
    (hrep : SqRep X d m n) :
    d = 1 ∨ d = 2

theorem n12_dual_two_squareclass_no_solution
    (m n W : ℤ) (hcop : Int.gcd m n = 1) :
    W ^ 2 ≠ 2 * m ^ 4 - 4 * m ^ 2 * n ^ 2 + 8 * n ^ 4

theorem Fdual_delta_image_exact :
    DeltaImageFdual = {1}
-/

end MazurProof.RationalPointsN12
```

The `n12_dual_two_squareclass_no_solution` lemma is the cleanest first local theorem to formalize. It is independent of the elliptic-curve API.

## 6. Rank / finite Mordell-Weil conclusion

The isogeny descent exact sequence gives

```text
#(F(ℚ) / 2F(ℚ)) = #δ(F(ℚ)) * #δ'(F'(ℚ)) = 4 * 1 = 4.
```

Since `F` has full rational 2-torsion,

```text
F(ℚ)[2] = {O, (-3,0), (0,0), (1,0)},
#F(ℚ)[2] = 4.
```

Thus the Mordell-Weil rank is `0`.

For Lean, I would not try to introduce a global rank theorem unless the project already has the necessary finitely-generated abelian-group infrastructure for rational points. There are two practical routes:

### Route A: group/rank route if API exists

Use the standard finite-generation/rank identity

```text
#(G/2G) = #(G[2]) * 2^rank(G)
```

for a finitely generated abelian group `G = F(ℚ)`. Then `#(F(ℚ)/2F(ℚ))=4` and `#F(ℚ)[2]=4` imply `rank=0`.

Interface sketch:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

/-
theorem F_two_torsion_exact :
    FTwoTorsion = {O, Pminus3, Pzero, Pone}

theorem F_mod_two_card_from_descent :
    FModTwoCard = 4

theorem F_rank_zero_from_descent :
    F_MW_rank = 0
-/

end MazurProof.RationalPointsN12
```

### Route B: self-contained height descent, probably shorter in this project

Avoid a rank API. Prove directly that every rational point is torsion using the explicit quotient enumeration.

The structure is:

```text
Every P ∈ F(ℚ) is congruent modulo 2F(ℚ) to one of
O, (0,0), (-1,2), (3,6)          -- representative choice.
```

Equivalently,

```text
P = R + 2Q
```

for one known representative `R` and some `Q`.

Then prove a specialized height descent lemma:

```text
If P is not one of the eight listed points and P = R + 2Q
with R in the representative set, then height_x(Q) < height_x(P).
```

Choose a counterexample with minimal denominator/naive height and descend. This avoids formalizing general Mordell-Weil rank. The height lemma is coordinate algebra and inequalities, but it is still less infrastructure than a general Mordell-Weil theorem.

Interface sketch:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

/-
def F_known_affine (X Y : ℚ) : Prop :=
  (X = -3 ∧ Y = 0) ∨
  (X = 0 ∧ Y = 0) ∨
  (X = 1 ∧ Y = 0) ∨
  (X = -1 ∧ (Y = 2 ∨ Y = -2)) ∨
  (X = 3 ∧ (Y = 6 ∨ Y = -6))

def xHeight (X : ℚ) : ℕ :=
  X.num.natAbs + X.den

theorem F_mod_two_representatives
    (P : FPointQ) :
    ∃ R Q : FPointQ,
      R ∈ {O, Tzero, Pminus, PplusT} ∧ P = R + 2 • Q

theorem F_height_drop_if_not_known
    {P R Q : FPointQ}
    (hP : P = R + 2 • Q)
    (hR : R ∈ {O, Tzero, Pminus, PplusT})
    (hnot : ¬ F_known_point P) :
    xHeight Q.X < xHeight P.X

theorem F_all_points_known_by_height_descent
    (P : FPointQ) : F_known_point P
-/

end MazurProof.RationalPointsN12
```

This is the route I would choose if `WeierstrassCurve` group/rank/reduction APIs are thin.

## 7. Torsion enumeration by reduction modulo 5

Once rank zero or the height descent gives that every point is torsion, enumerate torsion by good reduction at `p=5`.

The discriminant of

```text
x*(x-1)*(x+3)
```

is

```text
Δ = 16 * ((0-1)^2 * (0+3)^2 * (1+3)^2)
  = 16 * 1 * 9 * 16
  = 2304 = 2^8 * 3^2.
```

So `5` is a good prime. Torsion prime-to-`5`, and in fact all rational torsion, injects under reduction modulo `5` for good reduction.

Over `𝔽_5`, the affine solutions are exactly:

```text
(0,0), (1,0), (2,0), (3,1), (3,4), (4,2), (4,3),
```

plus the point at infinity. Thus

```text
#F(𝔽_5) = 8.
```

The eight rational points listed above reduce to these eight finite-field points. Therefore rational torsion has at most eight points and already has at least eight points, so the torsion subgroup is exactly the listed set.

A finite-field enumeration snippet:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

private def F5_on (x y : ZMod 5) : Prop :=
  y ^ 2 = x ^ 3 + 2 * x ^ 2 - 3 * x

example (x y : ZMod 5) (h : F5_on x y) :
    (x = 0 ∧ y = 0) ∨
    (x = 1 ∧ y = 0) ∨
    (x = 2 ∧ y = 0) ∨
    (x = 3 ∧ (y = 1 ∨ y = 4)) ∨
    (x = 4 ∧ (y = 2 ∨ y = 3)) := by
  fin_cases x <;> fin_cases y <;> norm_num [F5_on] at h ⊢

end MazurProof.RationalPointsN12
```

If `fin_cases` is brittle on `ZMod 5`, replace this by enumerating `Fin 5` representatives and coercing into `ZMod 5`.

Reduction-injectivity interface:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

/-
theorem F_good_reduction_at_5 : GoodReduction F 5

theorem F_torsion_reduction_injective_mod5
    {P Q : FPointQ}
    (hPtor : IsTorsion P)
    (hQtor : IsTorsion Q)
    (hred : reduceF5 P = reduceF5 Q) :
    P = Q

theorem F_torsion_points_exact :
    FTorsionQ = {O, (-3,0), (0,0), (1,0), (-1,2), (-1,-2), (3,6), (3,-6)}
-/

end MazurProof.RationalPointsN12
```

If the project does not yet have good-reduction injection, a specialized proof at `5` may be shorter than building a general theory. The proof is the standard formal-group argument: a torsion point reducing to `O` at a good prime has order a power of `5`; then compare with a second good prime such as `7`, or use the formal group over `ℤ_5` to rule it out. For this curve, using both `5` and `7` gives a very small finite bound because both groups have order `8`.

Over `𝔽_7`, one also gets `#F(𝔽_7)=8`; this is useful if the reduction-injection theorem is formulated prime-to-`p` only.

The desired theorem is then:

```lean

private def F_known_affine (X Y : ℚ) : Prop :=
  (X = -3 ∧ Y = 0) ∨
  (X = 0 ∧ Y = 0) ∨
  (X = 1 ∧ Y = 0) ∨
  (X = -1 ∧ (Y = 2 ∨ Y = -2)) ∨
  (X = 3 ∧ (Y = 6 ∨ Y = -6))

/-
theorem F_all_affine_points_known
    {X Y : ℚ} (hF : F_on X Y) :
    F_known_affine X Y
-/

/-- Target theorem, after unfolding `F_known_affine`. -/
/-
theorem F_N12_shifted_affine_points
    {X Y : ℚ}
    (hF : Y ^ 2 = X ^ 3 + 2 * X ^ 2 - 3 * X) :
    (X = -3 ∧ Y = 0) ∨
    (X = 0 ∧ Y = 0) ∨
    (X = 1 ∧ Y = 0) ∨
    (X = -1 ∧ (Y = 2 ∨ Y = -2)) ∨

  exact F_all_affine_points_known hF
```

## 9. Minimal Lean DAG

### Layer 0: curve arithmetic

```text
F_on, Fdual_on
point substitution checks
F discriminant = 2304
F5 finite-field enumeration
F7 finite-field enumeration, optional
```

### Layer 1: explicit isogenies

```text
φ_formula_maps_F_to_Fdual
φdual_formula_maps_Fdual_to_F
ker_φ = {O,(0,0)}
ker_φdual = {O,(0,0)}
φdual_comp_φ = double
φ_comp_φdual = double
```

For a first formalization pass, prove these as affine coordinate lemmas with explicit exceptional cases.

### Layer 2: descent maps

```text
δ_F definition by cases
δ_Fdual definition by cases
ker_δ_F = image φdual
ker_δ_Fdual = image φ
```

Avoid quotient API at first by representing squareclasses by squarefree integer reps.

### Layer 3: squareclass support

```text
F_sqclass_support:
  point on F, X ≠ 0, X=d*(m/n)^2 primitive squarefree
  -> d ∈ {±1,±3}

Fdual_x_nonneg:
  point on F' -> 0 ≤ X

Fdual_sqclass_support:
  point on F', X ≠ 0, X=d*(m/n)^2 primitive squarefree
  -> d ∈ {1,2}
```

### Layer 4: local obstruction

```text
n12_dual_two_squareclass_no_solution:
  gcd(m,n)=1 -> W^2 ≠ 2*m^4 - 4*m^2*n^2 + 8*n^4
```

This is pure integer arithmetic. Prove it first.

### Layer 5: image sizes

```text
δ(F(ℚ)) = {1,-1,3,-3}
δ'(F'(ℚ)) = {1}
#F(ℚ)/2F(ℚ) = 4
```

### Layer 6: rank/height conclusion

Choose one:

```text
rank route:
  Mordell-Weil finite generation + #G/2G formula -> rank 0

height route:
  every point P = R + 2Q with R known;
  if P not in final list, height(Q) < height(P);
  minimal counterexample contradiction.
```

The height route is likely more Lean-local.

### Layer 7: torsion enumeration

```text
known eight points are torsion
reduction mod 5 injects on torsion
#F(F_5)=8
known eight points reduce to all F(F_5)
therefore torsion points are exactly the known eight
```

### Layer 8: affine projection

```text
FPointQ finite set -> affine theorem over X,Y : ℚ
```

## 10. Mathlib/API pitfalls

1. **General elliptic-curve descent infrastructure may be absent or too abstract.** The explicit formulas for `φ`, `φ'`, `δ`, and the homogeneous spaces are short. A specialized proof is probably faster.

2. **Squareclass quotients over `ℚ*` can be expensive.** For the support and local obstruction lemmas, use squarefree integer representatives `X=d*(m/n)^2`. Introduce the quotient only after the finite support lemmas work.

3. **`WeierstrassCurve.Affine.Point` group law may not expose the formulas you want.** If simp does not unfold group law conveniently, define local affine addition/doubling formulas for this curve and prove agreement only where needed.

4. **Rank APIs may not exist for `F(ℚ)`.** Avoid a global rank theorem unless already available. A self-contained height descent from the explicit `mod 2` representatives may be shorter.

5. **Good-reduction injection may also be missing.** If so, either prove the specialized `p=5` injection lemma or use Nagell-Lutz as an alternative torsion enumeration. The reduction route is cleaner for this curve because `#F(𝔽_5)=8` and the known rational torsion already has size `8`.

6. **Finite-field enumeration over `ZMod p` is easy but tactic-sensitive.** `fin_cases` works in many Mathlib contexts; if it fails, switch to `Fin 5` and coerce.

7. **The dual squareclass `2` obstruction should be isolated.** It is pure `ℤ` arithmetic and should not depend on any elliptic-curve definitions.

## 11. If an arithmetic boundary is needed

The cleanest temporary boundary is not a rank axiom. Use the exact rational-point theorem for this curve:

```lean
import Mathlib

namespace MazurProof.RationalPointsN12

/-- Arithmetic boundary for the shifted N=12 curve. -/
/-
theorem F_N12_shifted_affine_points
    {X Y : ℚ}
    (hF : Y ^ 2 = X ^ 3 + 2 * X ^ 2 - 3 * X) :
    (X = -3 ∧ Y = 0) ∨
    (X = 0 ∧ Y = 0) ∨
    (X = 1 ∧ Y = 0) ∨
    (X = -1 ∧ (Y = 2 ∨ Y = -2)) ∨
    (X = 3 ∧ (Y = 6 ∨ Y = -6))
-/

end MazurProof.RationalPointsN12
```

But the route above shows this theorem is not Mazur-hard. It is an elementary 2-isogeny descent plus a finite mod-16 local obstruction and a finite torsion enumeration.
